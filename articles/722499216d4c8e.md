ITエンジニアを志す方々から、日々たくさんの相談をいただきます。
特に、異業種からこの世界へ挑戦しようとしている方からは、このような質問をよく受けます。

* **「どのプログラミング言語を身につければいいですか？」**
* **「まずはどんな資格を取るのが正解でしょうか？」**

特に年齢を重ねてからのキャリアチェンジとなると、「遠回りしたくない」「早く戦力になりたい」という切実な思いが痛いほど伝わってきます。

しかし、私はそうした質問に対し、あえて技術の話をする前に、いつも一つのアドバイスを送るようにしています。

## 技術や資格よりも先に、磨くべきもの
結論からお伝えしましょう。
エンジニアとして成功するために、技術的なスキルの心配よりもまず意識してほしいこと。

それは、**「主体性」を発揮すること**です。

「えっ、技術の話じゃないの？」と意外に思われるかもしれません。もちろん、コードが書けることや知識があることは重要です。しかし、技術や資格は、実はその後の問題だと私は考えています。

## 「主体性」とは？
「主体性」という言葉は、ビジネスの場でよく使われますが、似た言葉の「自主性」と混同されがちです。
スティーブン・R・コヴィー博士の著書の大ベストセラー『7つの習慣』において、「主体性（Proactivity）」は第1の習慣として掲げられており、すべての習慣の土台となる最も重要な概念です。

![](https://storage.googleapis.com/zenn-user-upload/d9bc3c25723e-20260411.png =150x)
*完訳 7つの習慣 30周年記念版（キングベアー出版）*

コヴィー博士は、主体性を単に「自分から進んで行動すること」以上の、非常に深い概念として定義しました。

『7つの習慣』において、主体性は「第1の習慣」として掲げられています。つまり、**すべての成功の土台となる最も重要なマインドセット**なのです。

### 1. 主体性の定義
主体性とは **「自分の人生に起こるすべての出来事を、他人のせいにせず、自分ごと（当事者意識）として捉えること」** です。
環境や他人のせいにするのではなく、「自分の人生のハンドルは自分が握っている」という自覚を持つことからすべてが始まります。

### 2. 選択の自由
コヴィー博士は、「責任（Responsibility）」という言葉を分解して、主体性の本質を説明しています。

* **Response（反応） ＋ Ability（能力）**
* つまり、**「自分の反応を選択する能力」** こそが主体性であると定義しました。

![](https://storage.googleapis.com/zenn-user-upload/8699478ac68e-20260413.png)
*反応的な行動と主体的な行動の違い*

私たちは日々、上司の叱責、突然のトラブル、嫌なニュースなど、さまざまな **「刺激」** を受けます。

* **主体的でない人（反応的な人）：** 刺激に対して感情のまま、反射的に反応します。「怒鳴られたから、嫌な気分になった。相手が悪い」と、自分の感情を相手に委ねてしまいます。
* **主体的な人：** 刺激を受けてから反応するまでの間に、**「選択の自由」** というスペースを確保します。

#### 具体例：上司に怒鳴られたとき
たとえば、理不尽に上司に怒鳴られたとしましょう。

主体的な人は、一呼吸おいて冷静に判断します。
「今、この人は感情的になっているな、気にせず自分の仕事に集中しよう」
「言い方はきついが、指摘自体は正しい。改善のチャンスにしよう」

このように、**「どう受け止めるか」を自分で選択できる人**は、感情やメンタルが安定し、常に前向きな一歩を踏み出すことができるのです。



### 3. 関心の輪と影響の輪
主体性を具体的な行動に移すためのツールが、**「関心の輪」と「影響の輪」** という考え方です。

![](https://storage.googleapis.com/zenn-user-upload/ecbfcf335de5-20260414.png)
*関心の輪と影響の輪*

* **関心の輪：** 自分が関心はあるが、**自分では変えられないこと**（天気、他人の性格、過去のミス、景気）。
* **影響の輪：** 自分の行動や考え方など、**自分で変えられること**（自分の態度、学び、今日何をするか）。

### どこにエネルギーを使っていますか？
多くの人は、自分ではどうしようもない「関心の輪」にエネルギーを浪費してしまいます。
* 「雨が降っているから、今日はやる気が出ない」
* 「上司の性格が変わればいいのに」
* 「老後の生活が不安で仕方ない」

これに対し、主体的な人は自分のエネルギーを **「影響の輪」** に集中させます。

* 「雨なら、家でじっくり読書をしてスキルアップしよう」
* 「上司の性格は変えられないが、報告の仕方を変えて信頼を得よう」
* 「老後のために、今月から1万円ずつ積立投資を始めよう」

**「自分が変えられること」に集中すると、不思議と「影響の輪」は少しずつ大きくなっていきます。**


## なぜ、「主体性」が重要か？
これまでの解説で、「主体性」という言葉の真意がかなりクリアになってきたのではないでしょうか。

ここで、冒頭でお話しした **「技術よりも先に、まずは主体性が重要である」** という理由に踏み込んでいきましょう。なぜ、スキルを磨く前にこのマインドセットが必要なのでしょうか？

それは、**主体性を発揮できる人こそが、周りから信頼され、チャンスを掴み取り、最速で成長できるから**です。

### 徹底比較！「主体性がある人」vs「ない人」

具体的にどのような違いがあるのか、行動のパターンを比較表にまとめました。自分自身や周りの人を思い浮かべながらチェックしてみてください。

| 主体的でない人（反応的） | 主体的な人 |
| :--- | :--- |
| 他人のせいにして逃げる | 自分事として一旦受け止める |
| 「自分は悪くない」と主張する | 何がいけなかったのかを考える |
| できない理由（言い訳）を探す | 次への反省と対策を練る |
| 「〜させられている」と受動的 | 「〜している」と能動的 |
| 自分の仕事以外はやらない | 厄介な問題（渦中の栗）も進んで拾う |
| 常に周囲にイライラしている | 常に平常心を保とうとする |
| 感情に任せて行動する | 考え抜いた上で行動を選択する |
| 変えられないことに不安になる | **自分にできること**に集中する |
| 自己中心的（クレクレ） | 他者貢献（ギブ） |
| 他人の目を気にして動く | **「正しいかどうか」** で動く |

### あなたなら、どちらの人と一緒に働きたいですか？

この表を見て、あなたはどちらのタイプと一緒にプロジェクトを成功させたいと思うでしょうか。答えは明白ですよね。

たとえ現時点で技術力が少し足りなかったとしても、主体性を発揮できる人であれば、リーダーはこう考えます。
**「この人なら、難しい仕事も責任を持ってやり遂げてくれるはず。よし、任せて育ててみよう」**

### 1. 「プラスのループ」に入る人
主体性がある人は、難しい仕事に挑戦する機会（チャンス）をどんどん手に入れます。
その結果、実力がつき、周囲からの信頼がさらに高まり、また新しい大きなチャンスが舞い込んでくる……という最高の **「成長のループ」** に入ることができます。

### 2. 「マイナスのループ」に陥る人
逆に、主体性がない人には「任せるのが怖い」ため、誰でもできるような単純作業しか回ってきません。
仕事にやりがいを感じられず、成長も止まり、さらに不満が溜まっていく……という **「停滞のループ」** に陥ってしまいます。

### 最後に：技術は「信頼」という土台の上に積み上がる
IT業界において、技術は確かに強力な武器です。しかし、その武器を使いこなすのは「人間」であり、仕事をお願いするのは「人」です。

**「この人に任せれば大丈夫だ」**

そう思わせる「主体性」という土台があってこそ、あなたが身につける技術や資格が、何倍もの価値を持って輝き始めます。

まずは今日、目の前の課題に対して「自分に何ができるか？」と一歩踏み出してみませんか？その小さな主体性の発揮が、あなたのキャリアを劇的に変える第一歩になるはずです。
