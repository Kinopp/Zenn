## 概要
**Prototype（プロトタイプ）パターン**は、**「新しいものを作るとき、ゼロから組み立てるのではなく、既存のものを『コピー（複製）』して作る」** という方法です。「たい焼き」を作るときに、毎回生地をこねて形を作るのではなく、「金型」を使って次々と複製していくイメージです。

## イメージ
【効果】
オブジェクトの生成過程にコストが掛かるかつ似たような複数のオブジェクトが必要な場合、効率的な実装ができる。

## メリット
### 1. オブジェクト作成のコストを抑えられる
クラスから `new` で新しくインスタンスを作る際、データベースへの問い合わせや複雑な計算が必要な場合があります。Prototypeパターンなら、**既にメモリ上にある完成品をコピーするだけ**なので、処理が非常に高速になります。

### 2. 複雑な「初期状態」をそのまま引き継げる
「設定項目が100個あり、そのうち90個は共通」というオブジェクトを作りたい場合、ゼロから作って90個設定するのは大変です。あらかじめ90個設定済みの「原型（プロトタイプ）」を用意しておけば、それをコピーして残り10個を変えるだけで済みます。

### 3. クラスの種類を増やさずに済む
「赤い弾」「青い弾」「速い弾」など、状態が少し違うだけのオブジェクトを大量に作る際、それぞれを別々のクラスにする必要がありません。1つの「弾クラス」のインスタンスを複数用意してコピーするだけで対応できます。



## デメリット
### 1. 「深いコピー（Deep Copy）」の罠
これが最大の難点です。オブジェクトが別のオブジェクト（リストや他のクラス）への参照を持っている場合、単にコピーすると「参照先（住所）」だけがコピーされ、**片方を変えるともう片方も変わってしまう**という事故が起きます。中身まで完全に複製する「深いコピー」の実装は非常に面倒でバグの温床になりやすいです。

### 2. `clone` メソッドの実装が必須
複製したいすべてのクラスに「自分をコピーする仕組み（`clone`など）」を実装しなければなりません。既存のライブラリのクラスなど、自分で中身を書き換えられないクラスには適用しにくいという制限があります。

### 3. 構造が複雑な場合の循環参照
「AがBを持っていて、BがAを持っている」というような循環参照がある場合、コピーの処理が無限ループに陥ったり、構造が破綻したりするリスクがあります。


## 適用パターン

| 項目 | 向いているケース |
| :--- | :--- |
| **生成コスト** | `new` して初期化するのに時間がかかる、または手順が複雑 |
| **バリエーション** | 状態が少しずつ違うオブジェクトが大量に必要 |
| **隠蔽** | 使う側が具体的なクラス名を知らなくても、コピーで増やしたい |

> **例え話で理解：細胞分裂**
> 
> 生物の細胞は、ゼロから新しい細胞を組み立てるのではなく、今ある自分の設計図をまるごとコピーして分裂します。
> プログラムの世界でも、**「全く同じものをもう一つ、手っ取り早く欲しい！」** というときにこの Prototype パターンが活躍します。

### 現場での使われ方
JavaScriptのような「プロトタイプベース」の言語では、言語の根幹にこの考え方が組み込まれています。JavaやC#などでは、ゲームの敵キャラクターを次々と生成（スポーン）させるような場面でよく使われます。


## サンプルソース
```Java:Prototype.java
public interface Prototype extends Cloneable{
    public Prototype createClone();
}
```
```Java:ConcretePrototype.java
package Prototype;

import java.util.logging.Level;
import java.util.logging.Logger;

public class ConcretePrototype implements Prototype{

    private String initialString = "";
    private String secondString = "";
    
    @Override
    public Prototype createClone() {
        Prototype prototype = null;
        try {
            prototype = (ConcretePrototype)clone();
        } catch (CloneNotSupportedException ex) {
            Logger.getLogger(ConcretePrototype.class.getName()).log(Level.SEVERE, null, ex);
        }
        return prototype;
    }
    
    public void setInitialString(String initialString){
        this.initialString = initialString;
    }
    
    public String getInitialString(){
        return initialString;
    }
    
    public void setSecondString(String secondString){
        this.secondString = secondString;
    }
    
    public String getSecondString(){
        return secondString;
    }
}
```
```Java:Client.java
public class Client {
    public static void main(String... args){
        // プロトタイプ作成
        ConcretePrototype concretePrototype = new ConcretePrototype();
        concretePrototype.setInitialString("初期値設定しました");
        System.out.println("--- 雛形作成 ---");
        System.out.println(concretePrototype.getInitialString());
        System.out.println(concretePrototype.getSecondString());

        System.out.println("--- 雛形をコピーし「１」を追加 ---");
        ConcretePrototype prototype1 = (ConcretePrototype)concretePrototype.createClone();
        prototype1.setSecondString("１");
        System.out.println(prototype1.getInitialString());
        System.out.println(prototype1.getSecondString());
        
        System.out.println("--- 雛形をコピーし「２」を追加 ---");
        ConcretePrototype prototype2 = (ConcretePrototype)concretePrototype.createClone();
        prototype2.setSecondString("２");
        System.out.println(prototype2.getInitialString());
        System.out.println(prototype2.getSecondString());
    }
}
```
