## 概要
**Iterator（イテレータ）パターン**を一言でいうと、**「中身がどう並んでいるか（配列、リスト、ツリーなど）を知らなくても、順番に1つずつ取り出せるようにする」** ためのパターンです。

わかりやすい例えは、**テレビのリモコンの「チャンネル送りボタン」** です。

* **テレビの中身：** 放送局のリストがメモリのどこに、どういう形式で保存されているかは視聴者には分かりません（複雑なデータ構造）。
* **リモコン：** 「次へ（Next）」というボタンがあるだけ。
* **メリット：** 視聴者は「次へ」を押すだけで、中身の仕組みを知らなくても全てのチャンネルを順番に見ることができます。

もしテレビのメーカーが、中身の保存方法を「配列」から「高度なデータベース」に変えたとしても、リモコンの「次へ」ボタンの使い方は変わりませんよね。これがIteratorパターンの本質です。

### 【解説】
### 1. Aggregate（集合体 / データの入れ物）
右側の緑のボックスです。
オブジェクトA、B、C…を詰め込んだ「宝箱」のようなものです。ここには **`iterator()`** というメソッドがあり、「私の数え方を知っている専用のガイド（Iterator）を一人派遣します」という機能を持っています。

### 2. Iteratorインターフェース（数え方の共通ルール）
左下の赤いボックスです。
どんなデータの集まりであっても、数えるときは必ずこの2つの質問（メソッド）を使います。
* **`hasNext()`:** 「次、まだある？」
* **`next()`:** 「じゃあ、次のやつ頂戴」
このシンプルなやり取りだけで、どんなに複雑なデータ構造でも中身を走査できます。

### 3. 既存のループ処理との違い（オレンジのボックス）
画像の中央にある `for` 文の例に注目してください。
従来の `for (int i=0; ...)` のような書き方だと、数える側（Client）が「今は何番目で、全部で何個あって…」と、**データの内部構造（配列なのか、長さはいくらか等）を詳しく知っている必要があります。**
Iteratorを使えば、Clientはデータの構造を一切知らなくて良くなるため、コードがスッキリし、バグも減ります。

### まとめ
この図のポイントは、**「数える手順を抽象化する」** ということです。
画像の下部にある通り、Javaなどの現代的な言語では言語標準の機能として取り込まれているため、自分で一から作る機会は少ないかもしれません。しかし、**「データの持ち主（Aggregate）」と「数える人（Iterator）」を分ける**という考え方は、美しく保守しやすいコードを書くための基本中の基本です。


## メリット
### 1. データ構造の「隠蔽」ができる（カプセル化）
使う側（クライアント）は、データが「配列」に入っているのか「連結リスト」なのか、あるいは「木構造」なのかを知る必要がありません。ただ「次はある？」「次をちょうだい」と聞くだけで済みます。
> これにより、後からデータ構造を「配列」から「より高速なリスト」に変更しても、**使う側のコードを1行も直さなくて済む**ようになります。

### 2. 数え上げのロジックを分離できる
「どうやって数えるか（順番通りか、逆順か、特定の条件のものだけか）」というロジックを、データを持つクラスから切り離せます。データクラスは「データの保持」に専念でき、**「単一責任の原則」** を守れます。

### 3. 同時に複数の「数え上げ」ができる
1つのリストに対して、複数の「数え係（Iterator）」を同時に走らせることができます。
* **例：** 「1番目から順に探す人」と「最後から順に探す人」が、お互いの現在地を邪魔することなく同じリストを同時にスキャンできます。



## デメリット
### 1. 単純な構造には「大げさ」すぎる
ただの配列をループ回すだけで良いような場面では、わざわざIteratorクラスを作るのは手間ですし、コード量も増えてしまいます。小規模なプログラムでは **「やりすぎ（オーバーエンジニアリング）」** になりがちです。

### 2. パフォーマンスのわずかな低下
直接インデックス（`i++`）でアクセスする場合に比べると、メソッド呼び出し（`next()`や`hasNext()`）を毎回挟むため、極限のスピードが求められる処理ではわずかなオーバーヘッドになります。

### 3. データの変更に弱い（種類による）
数え上げている途中で、元のリストから要素が削除されたり追加されたりすると、Iteratorが混乱してエラー（`ConcurrentModificationException`など）を出すことがあります。これを防ぐための実装は少し複雑になります。



## サンプルソース


```Java:Iterator.java
public interface Iterator {
    public boolean hasNext();
    public Object next();
}
```
```Java:ConcreteIterator.java
public class ConcreteIterator implements Iterator {
    private ConcreteAggregate aggregate;
    private int index;
    
    public ConcreteIterator(ConcreteAggregate aggregate){
        this.aggregate = aggregate;
        this.index = 0;
    }
    
    @Override
    public boolean hasNext() {
        return index < aggregate.getLength();
    }

    @Override
    public Object next() {
        Content content = aggregate.getContentAt(index);
        index++;
        return content;
    }
}
```
```Java:Aggregate.java
public interface Aggregate {
    public Iterator iterator();
}
```
```Java:ConcreteAggregate.java
import java.util.ArrayList;
import java.util.List;

public class ConcreteAggregate implements Aggregate{
    private List contents;
    
    public ConcreteAggregate(){
        this.contents = new ArrayList();
    }
    
    public Content getContentAt(int index){
        return (Content)contents.get(index);
    }
    
    public void appendContent(Content content){
        this.contents.add(content);
    }
    
    public int getLength(){
        return contents.size();
    }
    
    @Override
    public Iterator iterator() {
        return new ConcreteIterator(this);
    }
}
```
```Java:Content.java
public class Content {
    private String name;
    
    public Content(String name){
        this.name = name;
    }
    
    public String getName(){
        return name;
    }
}
```
```Java:Client.java
public class Client {
    public static void main(String... args){
        ConcreteAggregate concreteAggregate = new ConcreteAggregate();
        
        concreteAggregate.appendContent(new Content("１番目のコンテント"));
        concreteAggregate.appendContent(new Content("２番目のコンテント"));
        concreteAggregate.appendContent(new Content("３番目のコンテント"));

        Iterator iterator = concreteAggregate.iterator();
        while(iterator.hasNext()){
            Content content = (Content)iterator.next();
            System.out.println(content.getName());
        }
    }
}
```
