## 概要
**Memento（メメント）パターン**は、**「オブジェクトの『ある時点の状態』を保存しておき、後から元に戻せるようにする」** パターンです。
「メメント」とはラテン語で「形見」や「思い出」を意味します。一言でいえば、**強力な「やり直し（Undo）」機能**を実現するための仕組みです。

分かりやすい例えは、アクションゲームやRPGの **「セーブ機能」** です。

* **あなた（プレイヤー）：** 今のレベル、所持金、現在地などの複雑な状態を持っている。
* **セーブデータ：** その瞬間の状態をギュッと固めたデータ。中身はシステムにしか解読できない。
* **メモリーカード：** セーブデータを預かっておく場所。自分では中身をいじらない。

もしボス戦で負けてしまっても（状態が最悪になっても）、メモリーカードからデータを読み込めば、**「セーブした瞬間の状態」** に完全に巻き戻すことができますよね。

### 【解説】
### 1. Originator（創始者 / 作成対象）
画像右上の青いボックスです。
* **役割：** 「保存したい状態」を持っている本体です。
* **仕事：** 自分の今のスナップショットを **Memento** として作成したり、過去の **Memento** を渡されて元の状態に復元したりします。
* **ポイント：** 状態を一番よく知っているのは自分なので、自分でスナップショットを作ります。

### 2. Memento（思い出 / スナップショット）
画像右下のピンクのボックスです。
* **役割：** 特定の時点の状態（画像では2011年、2013年などの日付）を記録した **「情報のパッケージ」** です。
* **ポイント：** **「情報のブラックボックス」** であるべきです。後述するCaretakerには中身を見せず、Originatorだけが中身を理解できる仕組みにします。

### 3. Caretaker（世話人 / 管理者）
画像左上の青いボックスです。
* **役割：** Mementoをリストなどで **「預かっておくだけ」** の存在です。
* **仕事：** Originatorに「今の状態を保存して」と指示し、作られたMementoを受け取って保管します。また、必要に応じて「この時のMementoに戻して」と指示を出します。
* **ポイント：** 大切なのは、**Caretaker自身は保存されたデータの中身（状態）を一切いじらない（見られない）** という点です。

### まとめ
この図のキモは、**「直接状態をいじるのではなく、専用のパッケージ（Memento）を介してやり取りする」** という距離感にあります。
画像の中でCaretakerが掃除をしているイラストがありますが、まさに「身の回りの世話（保存のタイミング管理）」はするけれど、主人のプライベートな日記（中身の状態）は読まない、という絶妙な関係性を表していますね。


## メリット
### 1. カプセル化を壊さずに「スナップショット」が撮れる
通常、オブジェクトの状態を保存しようとすると、その内部データ（変数など）を外部に公開（getterなど）しなければなりません。しかし、Mementoパターンを使えば、**内部構造を秘密にしたまま**状態を外部（Caretaker）に預けることができます。

### 2. 「やり直し（Undo/Redo）」が簡単に実装できる
「今の状態を保存」してリストに入れておき、ユーザーが「戻したい」と言ったらリストから古いデータを取り出して適用するだけ。この履歴管理のロジックが非常にシンプルになります。

### 3. オブジェクトの責務が明確になる
状態を持っている「本人（Originator）」は、履歴の管理（何個保存するか、いつ捨てるか）を考える必要がありません。履歴管理は「管理者（Caretaker）」に任せられるため、**「単一責任の原則」** に則った綺麗な設計になります。



## デメリット
### 1. メモリを大量に消費する可能性がある
状態を保存するたびに新しいオブジェクト（Memento）を作るため、保存するデータのサイズが大きかったり、履歴を大量に残したりすると、**あっという間にメモリを使い果たしてしまいます。**

### 2. 処理に時間がかかる（オーバーヘッド）
状態をまるごとコピーして保存するため、頻繁にスナップショットを撮るような設計にすると、コピー処理自体がプログラムの動作を重くする原因になります。

### 3. 管理者（Caretaker）の負担
履歴を管理する側は、保存されたMementoの中身を見ることはできませんが、それらをいつ破棄するか、最大何個まで持つかといった「ライフサイクル」を適切に制御しなければなりません。

## 適用パターン

| 項目 | 向いているケース |
| :--- | :--- |
| **主な用途** | ユーザーによる「元に戻す（Undo）」機能の実装 |
| **データの性質** | 状態が複雑で、一箇所の変更が全体に波及するとき |
| **注意点** | メモリ使用量が増えすぎないよう、履歴の数に制限を設ける |

> **例え話で理解：ゲームの「セーブポイント」**
> * **プレイヤー（Originator）：** 今のレベルや持ち物を持っている。
> * **セーブデータ（Memento）：** その瞬間の状態を記録した小さなファイル。中身はプレイヤー本人にしか分からない。
> * **メモリーカード（Caretaker）：** 複数のセーブデータを保管している。中身は読めないが、「3つ目のデータをロードして」と言われたらプレイヤーに渡す。
> 
> もしボス戦で負けても、メモリーカードから「セーブデータ」を取り出せば、戦う前の状態に完全に戻れます。



## サンプルソース
```Java:Memento.java
public class Memento {
    private String state;
    
    public String getState() {
        return state;
    }

    public void setState(String state) {
        System.out.println("Mementoに" + state + "が設定されました");
        this.state = state;
    }
}
```
```Java:Caretaker.java
import java.util.ArrayList;
import java.util.List;

public class Caretaker {
    
    private Originator originator;
    private List<Memento> mementoList = new ArrayList<>();
    private List<Memento> redoList = new ArrayList<>();

    public Caretaker(Originator originator) {
        this.originator = originator;
    }

    public void createSnapShot(){
        // undoの作業が一旦切れるのでredoリストを一旦リセット
        if (!redoList.isEmpty()){
            redoList.clear();
        }
        this.mementoList.add(this.originator.createMemento());
    }
    
    public void undo(){
        int index = mementoList.size();
        if (index > 1){
            System.out.println("undo実施");
            redoList.add(mementoList.get(index-1));
            mementoList.remove(index-1);
            Memento memento = this.getLastMemento();
            originator.restoreMemento(memento);
        }
    }
    
    public void redo(){
        int index = redoList.size();
        if (index > 0){
            System.out.println("redo実施");
            Memento memento = this.getLastRedo();
            originator.restoreMemento(memento);
            redoList.remove(index-1);
        }
    }
    
    // 直近のMementoを取得
    private Memento getLastMemento(){
        int index = mementoList.size();
        if (index < 1){
            return null;
        }
        return mementoList.get(index-1);
    }
    
    // 直近のRedoListを取得
    private Memento getLastRedo(){
        int index = redoList.size();
        if (index < 0){
            return null;
        }
        return redoList.get(index-1);
    }
}
```
```Java:Originator.java
public class Originator {
    private String state;
    
    public Memento createMemento(){
        Memento memento = new Memento();
        memento.setState(state);
        return memento;
    }
    
    public void restoreMemento(Memento memento){
        this.state = memento.getState();
        System.out.println("Originatorに" + state + "が再設定されました");
    }

    public void setState(String state) {
        this.state = state;
        System.out.println("Originatorに" + state + "が設定されました");
    }
}
```
```Java:Client.java
public class Client {
    public static void main(String... args){
        Originator originator = new Originator();
        Caretaker caretaker = new Caretaker(originator);
        
        originator.setState("初期設定");
        caretaker.createSnapShot();    // 思い出作成
        
        originator.setState("修正１回目");
        caretaker.createSnapShot();    // 思い出作成
        
        originator.setState("修正２回目");
        caretaker.createSnapShot();    // 思い出作成
        
        caretaker.undo();
        caretaker.undo();
        
        caretaker.redo();
        caretaker.redo();
    }
}
```
