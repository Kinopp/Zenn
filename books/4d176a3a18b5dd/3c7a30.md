## 概要
**Interpreter（インタープリタ）パターン**は、**「独自の言語（ミニ言語）を作って、それを解釈・実行する」** ためのパターンです。特定の形式で書かれた文（数式や検索条件など）を解析し、プログラムが理解できる形に翻訳して実行します。

わかりやすい例えは、**「楽譜（スコア）」** です。

* **楽譜（命令）:** `ド・レ・ミ` という記号が並んでいます。
* **演奏家（インタープリタ）:** その記号を読み取り、「ドの音を鳴らす」「レの音を鳴らす」という実際の動作に変換します。

プログラムの世界では、「`1 + 2 - 3`」という文字列や、「`AかつB、またはC`」という検索条件などを、コンピュータが計算できる形に読み解く役割を担います。

### 【解説】
画像の上部にある「スタック」を使った計算の流れが、そのままインタープリタの動きになります。

### 1. 文法（Expressionインタフェース）
画像右下のピンクの枠です。
「数字」「プラス」「マイナス」といったすべての要素に共通する **`interpret(Stack)`** という窓口を作ります。これにより、数字だろうが記号だろうが、同じように「解釈して！」と命令できるようになります。

### 2. 各要素の「仕事」（具体的なクラス）
* **数字クラス：** 「自分（数字）をスタックに積む（push）」という仕事。
* **プラス・マイナスクラス：** 「スタックから2つ取り出して（pop）、計算して、結果を戻す（push）」という仕事。
このように、**一つの言葉（文字）に対して一つのクラスが責任を持つ**のがポイントです。

### 3. 2つのステップ（Parserの役割）
画像中央下の青い枠にある通り、2段階で動きます。
1.  **解析（文字列の解析）：** `1 2 - 4 5 + *` という文字列を見て、「これは数字クラス、これはマイナスクラス…」と、オブジェクトのリストに変換します。
2.  **実行（Interpret実行）：** そのリストを順番に叩いていくと、画像上部のスタック図のように、最終的な答え $$-9$$ が導き出されます。

### まとめ
この図のキモは、**「『式』そのものを、計算手順を知っているオブジェクトの集まりに変えている」** 点にあります。
画像の中でスタックの中身が $$(1-2) \times (4+5)$$ という一般式を解いていく様子が描かれていますが、インタープリタを使えば、プログラムに「この計算式を解け」と直接命じるのではなく、**「文法に従って自分たちで解決させる」**という賢いアプローチが可能になります。


## メリット

### 1. 文法（ルール）の変更や拡張が簡単
文法の各ルールがクラスとして独立しているため、新しいルールを追加したいときは新しいクラスを作るだけで済みます。既存のクラスを大幅に書き換える必要がないため、**拡張性が高い**のが特徴です。

### 2. 文法のコード化が直感的
バッカス・ナウア記法（BNF）などで定義された文法規則を、そのままクラス構造に落とし込むことができます。文法書を読みながらプログラムを組むような感覚で実装できるため、**ルールの再現性が高い**です。

### 3. 特定のドメインに特化した処理が得意
「特定の数式を計算する」「複雑な検索クエリを処理する」といった、限定的な範囲（ドメイン）での言語処理には非常に強力なツールとなります。


## デメリット

### 1. 複雑な文法には向かない（クラスの爆発）
文法のルールが増えれば増えるほど、クラスの数も膨大になります。例えば、一般的なプログラミング言語のような複雑な文法をこのパターンだけで作ろうとすると、管理が不可能なほど巨大なクラス群が出来上がってしまいます。
> ※複雑な言語処理には、専用のパーサジェネレータ（ANTLRやyacc/lexなど）を使うのが一般的です。

### 2. 実行速度が遅くなりやすい
このパターンでは、抽象構文木（AST）を再帰的に巡回して処理を行うため、非常に多くのメソッド呼び出しが発生します。単純なループ処理などに比べると、**パフォーマンス面では不利**です。

### 3. 初期の学習・実装コストが高い
文法を解析して「木構造」を組み立てる部分（パーサ）を自前で作る必要があり、そこには高度なプログラミング技術が求められます。

## 活用パターン

| 項目 | 向いているケース | 向いていないケース |
| :--- | :--- | :--- |
| **文法の複雑さ** | シンプルなルール（数式、正規表現など） | 複雑なプログラミング言語 |
| **優先順位** | 拡張性、保守性、ルールの明確さ | 実行速度、省メモリ |

### 具体的な活用例
* **数式の評価:** `( 1 + 2 ) * 3` のような文字列を計算する。
* **簡易スクリプト:** 特定のツール内だけで動く独自の自動化コマンド。
* **検索エンジン:** `A AND (B OR C)` のような検索条件の解析。



## サンプルソース
```Java:Expression.java
import java.util.Stack;

public interface Expression {
    public void interpret(Stack<Integer> stack);
}
```
```Java:TerminalExpressionMinus.java
import java.util.Stack;

public class TerminalExpressionMinus implements Expression{
    @Override
    public void interpret(Stack<Integer> stack) {
        stack.push( - stack.pop() + stack.pop() ); 
    }
}
```
```Java:TerminalExpressionMultiply.java
import java.util.Stack;

public class TerminalExpressionMultiply implements Expression {
    @Override
    public void interpret(Stack<Integer> stack) {
        stack.push( stack.pop() * stack.pop() ); 
    }
}
```
```Java:TerminalExpressionNumber.java
import java.util.Stack;

public class TerminalExpressionNumber implements Expression{
    private int number;
    
    public TerminalExpressionNumber(int number) {
        this.number = number;
    }
    
    @Override
    public void interpret(Stack<Integer> stack)  {
        stack.push(number);
    }
}
```
```Java:TerminalExpressionPlus.java
import java.util.Stack;

public class TerminalExpressionPlus implements Expression {
    @Override
    public void interpret(Stack<Integer> stack) {
        stack.push( stack.pop() + stack.pop() ); 
    }
}
```
```Java:Parser.java
import java.util.ArrayList;
import java.util.List;
import java.util.Stack;

public class Parser {
    private List<Expression> parseTree = new ArrayList<Expression>();
 
    public Parser(String s) {
        for (String token : s.split(" ")) {
            if (token.equals("+")){
                parseTree.add( new TerminalExpressionPlus() );
            } else if (token.equals("-")) {
                parseTree.add( new TerminalExpressionMinus() );
            } else if (token.equals("*")) {
                parseTree.add( new TerminalExpressionMultiply() );
            } else {
                parseTree.add( new TerminalExpressionNumber(Integer.valueOf(token)) );
            }
        }
    }
 
    public int evaluate() {
        Stack<Integer> context = new Stack<Integer>(); 
        for (Expression e : parseTree){
            e.interpret(context);
        }
        return context.pop();
    }
}
```
```Java:Client.java
public class Client {
    public static void main(String[] args){
        String expression = "1 2 - 4 5 + *";
        Parser parser = new Parser(expression);
        System.out.println("'" + expression +"' equals " + parser.evaluate());
    }
}
```
