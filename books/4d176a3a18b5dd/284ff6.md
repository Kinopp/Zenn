## 概要
Mediatorは「仲裁者」という意味です。**Mediator（メディエーター）パターン**は、**「バラバラに動くオブジェクトたちの中心に立ち、連絡係を引き受ける」** パターンです。

たくさんのオブジェクト（部品）がお互いに直接メッセージを送り合うと、網の目のように複雑な関係になってしまいます。Mediatorパターンでは、各オブジェクトは **「仲裁者（Mediator）」にだけ報告し、指示もそこから受ける** ようにします。

分かりやすい例えは、**飛行機と航空管制塔**の関係です。

* **もし管制塔がなかったら：**
    すべてのパイロットが他のすべてのパイロットと直接無線で話し、「今から着陸するよ！」「え、私も！」「どいて！」とやり取りしなければなりません。飛行機の数が増えると、通信網はスパゲッティのように絡まり、大混乱に陥ります。
* **管制塔（Mediator）がある場合：**
    パイロットは**管制塔だけに**報告し、指示を仰ぎます。他の飛行機がどこにいるかを知る必要はありません。管制塔が全体を俯瞰して、「A機は着陸、B機は旋回待機」と整理してくれるからです。

### 【解説】
### 1. 登場人物（同僚：Colleagues）
画像の中の「名前入力テキスト」「住所入力テキスト」「送信ボタン」がこれに当たります。彼らは自分の仕事（文字が入った、クリックされた）は分かりますが、他の人の状況は知りません。

### 2. 調停者（Mediator）
真ん中の「Mediator」が司令塔です。
* テキスト欄に文字が入ると、「入力がありました！」とMediatorに報告します。
* Mediatorは「名前も住所も埋まったな」と判断し、送信ボタンへ **「送信していいよ！」** と合図を送ります。

### 3. 直接参照しない！（赤い×印）
ここがこのパターンの肝です。
もしボタンが直接テキスト欄を見に行こうとすると、**「ボタンはテキスト欄の存在を知っていなければならない」**という依存関係が生まれます。項目が増えるたびに、ボタンのコードを書き直すのは大変ですよね？
Mediatorがいれば、各パーツは**「Mediatorに報告するだけ」** で済むので、パーツ同士をバラバラに入れ替えるのが簡単になります。

### まとめ
この図は、**「オブジェクト間の複雑な結びつきをバラバラにほどき、一箇所で管理する」** ことの重要性を説いています。
「送信ボタンが押せるかどうか」という判断ロジック（ビジネスロジック）をボタン自身に持たせず、Mediatorという第三者に預けることで、画面全体の制御をスッキリさせているのです。


## メリット
### 1. 「多対多」の関係を「1対多」に整理できる
各オブジェクト（Colleague）は他のオブジェクトを誰一人知る必要がありません。知っているのは「仲裁者」だけ。これにより、オブジェクト間の依存関係（結合度）が劇的に下がり、コードの見通しが良くなります。

### 2. 再利用性が高まる
特定のオブジェクトが他の特定のオブジェクトに依存していないため、その部品を別の場所へ持っていって再利用するのが簡単になります。

### 3. 相互作用のロジックを1箇所にまとめられる
「Aが変化したらBとCを更新する」といった、複雑な連携ルールが仲裁者クラスの中に集約されます。挙動を変えたいときは、仲裁者のコードだけを修正すれば済みます。



## デメリット
### 1. 仲裁者が「神（ゴッドオブジェクト）」になりがち
すべての通信を1つのクラスで管理するため、システムが大規模になると**仲裁者クラスが肥大化し、非常に複雑で巨大なコード**になってしまいます。これが最大のデメリットです。

### 2. 仲裁者の保守が難しくなる
「何でも知っている」クラスは、どこか一箇所を直すと全体に影響が出るリスクが高まります。本来バラバラにしておきたかった複雑さが、仲裁者という一点に凝縮されてしまうのです。

## 活用パターン

| 項目 | 向いているケース |
| :--- | :--- |
| **関係性** | 多くのオブジェクトが複雑に絡み合っているとき |
| **再利用性** | 各部品を独立させて、他のプロジェクトでも使いたいとき |
| **保守性** | 連携ロジックがあちこちに散らばっていて修正が辛いとき |

> **例え話で理解：航空管制塔**
> 飛行機（パイロット）同士が直接「今から着陸するよ！」「ちょっと待って！」と無線でやり取りしていたら大混乱になりますよね。
> そこで **管制塔（Mediator）** が登場します。パイロットは管制塔にだけ報告し、管制塔が全体を見て「あなたは着陸、あなたは待機」と指示を出します。これにより、安全でスムーズな運行が可能になります。



## サンプルソース


```Java:Mediator.java
public interface Mediator {
    public void addColleagues(Colleague colleague);
    public void colleagueChanged();
}
```
```Java:ConcreteMediator.java
import java.util.ArrayList;
import java.util.List;

public class ConcreteMediator implements Mediator{

    private List<Colleague> colleagueList = new ArrayList();
    
    @Override
    public void addColleagues(Colleague colleague) {
        colleagueList.add(colleague);
    }

    @Override
    public void colleagueChanged() {
        int unUsedcount = 0;
        for(Colleague colleague : colleagueList){
            if (colleague.getStatus().equals("未使用")){
                unUsedcount += 1;
            }
        }
         
        String message = null;
        if(unUsedcount == 0){  //「未使用」が１つもない場合
            message = "もういっぱいです";
        } else {               //「未使用」がある場合
            message = "まだ使えます";
        }
        // Colleagueへメッセージを渡す
        for(Colleague colleague : colleagueList){
            colleague.controlColleague(message);
        }
    }
}
```
```Java:Colleague.java
public abstract class Colleague {
    private Mediator mediator;
    private String status;
    
    public String getStatus(){
        return this.status;
    }
    
    public void setStatus(String status){
        this.status = status;
    }
    
    public void setMediator(Mediator mediator){
        this.mediator = mediator;
    }
    protected abstract void controlColleague(String message);
}
```

```Java:Client.java
public class Client {
    public static void main(String... args){
        
        Colleague colleague11 = new ConcreteColleague1();
        Colleague colleague12 = new ConcreteColleague1();
        Colleague colleague21 = new ConcreteColleague2();
        Colleague colleague31 = new ConcreteColleague3();
        
        // 1回目の判定（１つだけ未使用あり）
        Mediator mediator1 = new ConcreteMediator();

        colleague11.setStatus("未使用");
        colleague12.setStatus("使用済");
        colleague21.setStatus("使用済");
        colleague31.setStatus("使用済");
        
        mediator1.addColleagues(colleague11);
        mediator1.addColleagues(colleague12);
        mediator1.addColleagues(colleague21);
        mediator1.addColleagues(colleague31);
        
        mediator1.colleagueChanged();
        
        // 2回目の判定（未使用なし）
        Mediator mediator2 = new ConcreteMediator();
        
        colleague11.setStatus("使用済");
        colleague12.setStatus("使用済");
        colleague21.setStatus("使用済");
        colleague31.setStatus("使用済");
        
        mediator2.addColleagues(colleague11);
        mediator2.addColleagues(colleague12);
        mediator2.addColleagues(colleague21);
        mediator2.addColleagues(colleague31);
        
        mediator2.colleagueChanged();
    }
}
```
