## 概要
Singleton（シングルトン）パターンは、一言でいうと **「世界にたった一つだけの特別な存在（インスタンス）を保証する」** 仕組みのことです。
たとえるなら、**「一国における一人の大統領」** や **「一つの会社における一人の社長」** のようなイメージです。あちこちに大統領がいたら命令がバラバラになって困りますよね？ それをプログラミングで解決するのがこのパターンです。

#### 【ポイント】
Singletonパターンにおいて、**インスタンスの重複生成を物理的に不可能にする**ための3つのステップは以下の通りです。

#### 1. コンストラクタの隠蔽（Private Constructor）
クラス外部からの `new` 演算子による直接的なインスタンス化を禁止します。コンストラクタを **private** に設定することで、開発者が意図せず複数のオブジェクトを作ってしまうミスを根本から防ぎます。

#### 2. 内部での唯一のインスタンス管理（Static Instance）
外部からの生成を封じる代わりに、クラス内部で**自分自身のインスタンスを一度だけ生成**し、静的（static）な変数として保持します。これにより、システム全体で「たった一つの個体」が常に維持されるようになります。

#### 3. グローバルなアクセスポイントの提供（getInstance）
内部で保持している唯一のインスタンスを、外部から安全に取得するための窓口（メソッド）を用意します。Javaの世界では、このメソッドを `getInstance()` と命名するのが標準的なルールとなっています。

> この「3つの工夫」が揃うことで、初めて「型」としての安全性が保証されます。単に「1つだけ使うように気をつける」という運用ルールではなく、**「システム的に1つしか作れない構造にする」** のがデザインパターンの真髄です。


## メリット
### 1. メモリと資源の節約
同じ役割のオブジェクトを、使うたびに「new（新規作成）」していると、メモリをどんどん消費してしまいます。Singletonなら**最初に作った1つを使い回す**ので、とてもエコです。

### 2. 「唯一の窓口」になれる（データの一貫性）
たとえば「システムの設定情報」や「ログ出力」を管理する場合、複数のオブジェクトが勝手にバラバラな設定を持っていたら困りますよね。
* **メリット:** どこからアクセスしても**「常に同じ最新の状態」**を見ることができます。

### 3. どこからでも呼べる（グローバルなアクセス）
「あの設定、どこに置いたっけ？」と探す必要がありません。`Singleton.getInstance()` のように決まった呼び出し方をするだけで、いつでもその「唯一の存在」に会いにいけます。

## デメリット
### 1. テストがしにくい（最大の弱点！）
プログラミングでは、機能ごとに切り分けてテスト（ユニットテスト）をするのが基本です。しかし、Singletonは「状態」をずっと保持し続けてしまうため、**「Aのテストの結果が、Bのテストに影響を与えてしまう」**という問題が起きやすいです。

### 2. 「密結合」になりやすい
どこからでも呼べるということは、**「そのSingletonがないと動かないプログラム」** が量産されるということです。後から「やっぱり2つに分けたい」「別の機能に差し替えたい」と思ったときに、修正が非常に大変になります。

### 3. マルチスレッドでの事故
複数の処理が同時に「getInstance（作って！）」とお願いすると、タイミングによっては **「一瞬だけ2人存在してしまう」** というバグが起きることがあります。これを防ぐには少し複雑なコードを書く必要があります。

## 適用パターン
Singletonは **「便利だけど、依存しすぎると後が怖い劇薬」** のような存在です。

| 項目 | 内容 |
| :--- | :--- |
| **向いているもの** | 設定管理、ログ出力、キャッシュ、データベース接続の管理 |
| **向いていないもの** | 頻繁に状態が変わるもの、テストを細かく行いたい機能 |



## サンプルソース
```Java:Singleton.java
public class Singleton {
    private static final Singleton SINGLETON = new Singleton();

    // コンストラクタをprivateにすることでインスタンス化および継承を禁止する。
    private Singleton() {                                 
    }

    public static synchronized Singleton getInstance() {
        // このメソッドを通してのみインスタンスを取得できる。
        // staticなインスタンスなのでどこから呼ばれても常に同じオブジェクトを渡す。
        // 同時実行を考慮し、synchronizedとする
        return SINGLETON;
    }
}
```

```java:Client.java
public class Client {
    public static void main(String... args){
        Singleton singleton01 = Singleton.getInstance();
        Singleton singleton02 = Singleton.getInstance();
        
        // インスタンスを２回取得して、同じものであるかの検証
        if (singleton01 == singleton02){
            System.out.println("同じオブジェクトです");
        } else {
            System.out.println("違うオブジェクトです");
        }
    }
}

```
