## 概要
**Facade（ファサード）パターン**は、**「複雑な内部処理を、一つのシンプルな窓口にまとめる」** 仕組みのことです。
フランス語の「Facade」は建物の「正面（顔）」を意味します。
たとえるなら、**「ホテルのコンシェルジュ」** や **「パソコンの電源ボタン」** のようなイメージです。
パソコンの中では、CPUの初期化、メモリのチェック、OSの読み込み……といった非常に複雑なことが起きていますが、使う人は「電源ボタン」を一つ押すだけで済みます。この「ボタン」がFacadeの役割です。

## イメージ
利用者側は窓口を経由してのみ対象システムにアクセスするのでシステム内部の構造を意識することなく利用できる。
![](https://storage.googleapis.com/zenn-user-upload/ccd3d35ddde5-20260401.png =580x)



## メリット
### 1. 使う側（Client）がとても楽になる
サブシステムの中に10個のクラスがあっても、使う人はFacadeクラスのメソッドを1つ呼ぶだけで済みます。「どのクラスをどの順番で呼べばいいか」という難しいルールを知らなくていいので、使い勝手が劇的に向上します。

### 2. サブシステムとの「つながり」を弱くできる（疎結合）
もし内部のクラス（サブシステム）の仕様が変わっても、Facadeがその変化を吸収してくれれば、それを利用しているプログラムを修正する必要はありません。システムの変更に強い構造になります。

### 3. 学習コストを下げられる
新しい開発者がチームに入ったとき、複雑な内部構造をすべて理解しなくても、まずは「Facadeの使い方」さえ覚えれば作業が進められるようになります。



## デメリット
### 1. Facade自体が巨大化しやすい（神オブジェクト化）
あれもこれもと便利にしようとして、Facadeクラスにすべての処理を詰め込みすぎると、そのクラスだけが何千行もある「巨大で不透明なクラス」になってしまう危険があります。

### 2. 内部の細かい制御ができなくなる（こともある）
Facadeは「よくある処理」を簡単にするためのものなので、特定のケースで「内部のこのクラスの、この設定だけを細かく変えたい！」というマニアックな要望には応えにくいことがあります。
（※もちろん、内部クラスを直接触ることを禁止しなければ解決しますが、ルールが曖昧になりがちです）

### 3. 結局、内部を知らないとデバッグできない
表面上はシンプルに見えても、エラーが起きたときは結局Facadeの裏側にある複雑な迷宮に潜り込んで調査しなければなりません。「Facadeがあるから中身は適当でいい」というわけではないのが難しいところです。

## 適用パターン
| おすすめの用途 | 避けるべきケース |
| :--- | :--- |
| **複雑なライブラリを簡単に使わせたいとき** | 内部のクラスが1〜2個しかなく、すでにシンプルなとき |
| **古いシステム（レガシー）を隠して、綺麗な窓口を作りたいとき** | 使う人が常に内部の細かい設定をいじる必要があるとき |
| **複数のクラスが連携して動く「決まった手順」があるとき** | 処理の全容が頻繁に変わり、窓口を固定できないとき |


## サンプルソース
```Java:Facade.java
public class Facade {
    
    // インスタンスの生成は不要
    private Facade(){
    }
    // 外部との窓口になるメソッド
    public static void facadeMethod(){
        ClassA classA = new ClassA();
        ClassB classB = new ClassB();
        ClassC classC = new ClassC();
        
        classA.methodA();
        classB.methodB();
        classC.methodC();
    }
}
```
```Java:ClassA.java
public class ClassA {
    protected void methodA(){
        System.out.println("methodAです");
    }
}
```

```Java:ClassB.java
public class ClassB {
    protected void methodB(){
        System.out.println("methodBです");
    }
}
```
```Java:ClassC.java
public class ClassC {
    protected void methodC(){
        System.out.println("methodCです");
    }
}
```

```Java:Client.java
public class Client {
    public static void main(String... args){
        Facade.facadeMethod();
    }
}
```


### 実行結果
```
methodAです
methodBです
methodCです
```
