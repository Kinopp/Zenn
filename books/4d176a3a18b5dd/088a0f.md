## 概要
**Adapter（アダプター）パターン**を一言でいうと、**「インターフェースのズレを埋めて、つながらないもの同士をつなげる」** パターンです。別名「Wrapper（ラッパー）パターン」とも呼ばれます。

一番わかりやすい例えは、**海外旅行用の電源変換プラグ**です。
* **あなたが持っているもの：** 日本の家電（コンセントの形は「平型 2ピン」）。
* **訪れた国のコンセント：** 海外の壁（形は「丸型 3ピン」）。
* **問題：** 形が違うので、そのままでは差し込めない！
* **解決策：** 間に「変換アダプター」を挟む。

この「変換アダプター」が、まさにAdapterパターンの役割です。**「日本の家電（既存のクラス）」も「海外の壁（新しいシステム）」も一切改造することなく**、間につなぎ役を入れるだけで動くようになります。

### 【解説】
### 1. Targetインターフェース（今のルール）
図の右上にある、Clientが直接見ている窓口です。
「気圧は `getPressureForPascal()`（パスカル）という名前で取得する」という**新しい時代のルール**が定義されています。

### 2. Adaptee（古いけれど使える遺産）
図の中央下にある「既存気象衛星システム」です。
こいつは非常に優秀ですが、古い設計なので「気圧は `getPressureForBar()`（バール）で返す」という**古いルール**で動いています。
そのままでは、パスカルを欲しがっているClientとは話が噛み合いません。

### 3. Adapter（通訳者）
ここがこのパターンの心臓部です。
1. Clientから `getPressureForPascal()` と呼ばれる。
2. 裏側でこっそり古いシステムの `getPressureForBar()` を呼ぶ。
3. 得られた「バール」の値を、計算して「パスカル」に**変換**する。
4. 何食わぬ顔でClientにパスカルの値を返す。

これにより、Clientは相手が古いシステムであることを**一切気にせずに**、新しいルールで通信できるようになります。

### まとめ
この図のポイントは、**「既存システム（Adaptee）に一切手を加えていない」** という点です。
「古いから作り直そう」ではなく、「アダプターを被せて今のルールに合わせよう」という考え方は、大規模なシステム開発や、外部ライブラリを導入する際に非常に重宝します。


## メリット
### 1. 既存のクラスを修正せずに再利用できる
これが最大のメリットです。既にテスト済みで安定している古いクラスや、中身を触ることができない外部ライブラリを、**一切変更することなく**新しいシステムに組み込めます。

### 2. 「新しい仕組み」と「古い仕組み」を切り離せる
本体のプログラムは「アダプター」という窓口だけを知っていればよく、その奥に潜む「古いクラス」の具体的な書き方を気にする必要がありません。

### 3. バグのリスクを抑えられる
既存のコードに手を加えないため、修正による先祖返りや、思わぬところへの悪影響を防ぐことができます。



## デメリット
### 1. クラスの数が増えて複雑になる
本来やりたかった処理にたどり着くまでに「アダプター」という中間層を通るため、クラスの数が増え、構造が一段階複雑になります。

### 2. 処理のオーバーヘッド
メソッドを呼び出す際にアダプターを中継するため、直接呼び出す場合に比べると（無視できるレベルではありますが）微小な実行時間のロスが発生します。

### 3. 本質的な解決にならないこともある
アダプターで無理やりつないでいるだけで、古いクラスが抱えている技術的な負債そのものを解消するわけではありません。あまりに多用すると、システムのあちこちが「変換器だらけ」になり、全体像が把握しにくくなります。



## サンプルソース


```Java:Target.java
public interface Target {
    public void newMethod1();
    public void newMethod2();
}
```
```Java:Adapter.java
public class Adapter implements Target{
    private Adaptee adaptee;
    
    public Adapter(){
        this.adaptee = new Adaptee();
    }

    @Override
    public void newMethod1() {
        adaptee.oldMethod1();
    }

    @Override
    public void newMethod2() {
        adaptee.oldMethod2();
    }
}
```
```Java:Adaptee.java
public class Adaptee {
    public void oldMethod1(){
        System.out.println("旧メソッド１です");
    }
    
    public void oldMethod2(){
        System.out.println("旧メソッド２です");
    }
}
```
```Java:Client.java
public class Client {
    public static void main(String... args){
        Target target = new Adapter();
        target.newMethod1();
        target.newMethod2();
    }
}
```


### 実行結果
```
旧メソッド１です
旧メソッド２です
```
