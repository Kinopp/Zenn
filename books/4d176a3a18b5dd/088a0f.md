## イメージ
既存モジュールを有効活用したい場合、既存モジュールのインタフェースを新システムのインタフェースに変換して使用可能となる

【事例】
気象衛星システムを新たに導入。新システムは気圧を取得するメソッドgetPressureForPascal()を提供しておりパスカルで返却する。
一部の部署では既存の気象衛星システムを導入しており、気圧を取得するメソッドgetPressureForBar()はバールで返却する。
利用側には新旧システムの差異を意識させないよう、外部にはTargetインタフェースしか見せないようにし、旧システムは新システムに合わせるようAdapterで単位変換する。

## サンプルソース


```Java:Target
public interface Target {
    public void newMethod1();
    public void newMethod2();
}
```
```Java:Adapter
public class Adapter implements Target{
    private Adaptee adaptee;
    
    public Adapter(){
        this.adaptee = new Adaptee();
    }

    @Override
    public void newMethod1() {
        adaptee.oldMethod1();
    }

    @Override
    public void newMethod2() {
        adaptee.oldMethod2();
    }
}
```
```Java:Adaptee
public class Adaptee {
    public void oldMethod1(){
        System.out.println("旧メソッド１です");
    }
    
    public void oldMethod2(){
        System.out.println("旧メソッド２です");
    }
}
```
```Java:Client
public class Client {
    public static void main(String... args){
        Target target = new Adapter();
        target.newMethod1();
        target.newMethod2();
    }
}
```


### 実行結果
```
旧メソッド１です
旧メソッド２です
```
