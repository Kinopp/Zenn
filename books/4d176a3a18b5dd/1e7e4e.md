## 概要
**Observer（オブザーバー）パターン**は、**「誰かが変わったら、登録メンバー全員に自動で通知する」** パターンです。別名「出版-購読（Publish-Subscribe）モデル」とも呼ばれます。

【効果】
SubjectとObserverの変更をお互い意識せずに行える

## メリット
### 1. 疎結合（ルーズカップリング）を実現できる
通知を送る側（Subject）は、通知を受け取る側（Observer）が「具体的に誰なのか」「何をするのか」を知る必要がありません。ただ「リストに入っている人たちに通知を送る」という仕事に専念できます。

### 2. 放送通信（ブロードキャスト）が簡単
1つの変化に対して、複数のオブジェクトを同時に動かすことができます。
* **例：** ユーザーが設定を変更したら、「画面の背景色を変えるクラス」「ログを記録するクラス」「サーバーに通知するクラス」が一斉に動き出す、といった処理がスマートに書けます。

### 3. リアルタイムな状態同期
データが更新された瞬間に自動で通知が行くため、常に最新の状態を保つことができます。自分で何度も「更新された？」と聞きに行く（ポーリング）必要がなくなります。



## デメリット
### 1. 通知の順番やタイミングが制御しにくい
基本的には「登録されている順」などで通知が行きますが、特定のObserverが処理に時間をかけると、全体のパフォーマンスに影響したり、予期せぬ順序で画面が更新されたりすることがあります。

### 2. 「連鎖反応」によるカオス
Aの変化がBに通知され、Bの変化がCに通知され……という連鎖が起きると、どこで何が起きているのか把握するのが困難になります。最悪の場合、**無限ループ**に陥る危険もあります。

### 3. メモリリーク（放置されたリスナー）の原因になる
Observerを登録（購読）したまま削除し忘れると、SubjectがそのObserverへの参照を持ち続けてしまい、ガベージコレクションで解放されなくなります。これは「Lapsed Listener問題」と呼ばれる有名なバグの原因です。


## 適用パターン

| 項目 | 向いているケース |
| :--- | :--- |
| **用途** | 1つのオブジェクトの状態変化を、他の多くのオブジェクトに伝えたいとき |
| **代表例** | SNSのフォロー通知、株価チャートの更新、ボタンクリックなどのイベント処理 |
| **注意点** | 使い終わったら必ず「登録解除（Unsubscribe）」を行うこと |

> **例え話で理解：YouTubeのチャンネル登録**
> * **YouTuber（Subject）：** 新しい動画を投稿した（状態が変わった）ら、通知ボタンを押す。
> * **視聴者（Observer）：** 登録している人全員に、スマホの通知が届く。
> 
> YouTuberは、視聴者一人ひとりの住所や名前を知らなくても、登録者リストに「送る」だけで済みます。視聴者は、見たくなければ「登録解除」すればいいだけ。非常に自由度の高い関係ですよね。


## サンプルソース


```Java:Observer.java
public interface Observer {
    public void update();
}
```
```Java:Subject.java
import java.util.ArrayList;
import java.util.List;

public abstract class Subject {
    private List<Observer> observers = new ArrayList<>();
    
    public void addObserver(Observer observer){
        observers.add(observer);
    }
    
    public void notifyObservers(){
        getSubjectStatus();
        for(Observer observer : observers){
            observer.update();
        }
    }
    
    public abstract void getSubjectStatus();
}
```
```Java:ConcreteObserver1.java
public class ConcreteObserver1 implements Observer{
    @Override
    public void update() {
        System.out.println("ConcreteObserver1はSubjectの変更通知を受けました");
    }
}
```
```Java:ConcreteObserver2.java
public class ConcreteObserver2 implements Observer{
    @Override
    public void update() {
        System.out.println("ConcreteObserver2はSubjectの変更通知を受けました");
    }
}
```
```Java:ConcreteSubject.java
public class ConcreteSubject extends Subject{
    @Override
    public void getSubjectStatus() {
        System.out.println("ConcreteSubjectに変更が発生しました");
    }
}
```
```Java:Client.java
public class Client {
    public static void main(String... aegs){
        Subject subject = new ConcreteSubject();
        Observer observer1 = new ConcreteObserver1();
        Observer observer2 = new ConcreteObserver2();
        
        subject.addObserver(observer1);
        subject.addObserver(observer2);
        subject.notifyObservers();
    }
}
```


### 実行結果
```
ConcreteSubjectに変更が発生しました
ConcreteObserver1はSubjectの変更通知を受けました
ConcreteObserver2はSubjectの変更通知を受けました
```
