## 概要
**Builder（ビルダー）パターン**は、**「複雑なモノの組み立て工程を、専門の職人（Builder）に任せる」** パターンです。
特に、設定項目（引数）が多すぎて「どの数字が何を意味しているのか分からない！」という混乱を防ぐために非常に重宝されます。
オブジェクトを作るとき、普通は `new` を使ってコンストラクタを呼びますが、引数が10個もあると順番を間違えたり、意味を読み違えたりします。Builderパターンでは、**「1つずつ部品をセットしていき、最後に『完成！』と指示を出す」** というステップを踏みます。

## メリット
### 1. コードの可読性（読みやすさ）が劇的に上がる
メソッドチェーン（`.name("名前").age(20).build()` のようにつなげる書き方）が使えるため、どの値が何の項目にセットされているかが一目でわかります。
> **例:** `new User("田中", 20, "男", true, false...)` よりも、 `User.builder().name("田中").age(20).isMember(true).build()` の方が圧倒的に分かりやすいですよね。

### 2. 「不完全な状態」のオブジェクトを作らせない
全ての部品が揃うまで「完成（build）」させない仕組みにできるため、作りかけの壊れたオブジェクトがシステム内に紛れ込むのを防げます。また、完成した後は中身を変えられない **「不変（Immutable）」** なオブジェクトも作りやすくなります。

### 3. 同じ工程で「違うバリエーション」が作れる
「豪華な家を建てる職人」と「安い家を建てる職人」を切り替えるだけで、同じ組み立て手順（壁を作る→屋根を乗せる）のまま、全く異なる完成品を作ることができます。



## デメリット
### 1. コードの量が増える（ボイラープレート）
本来作りたいクラスの他に、「Builder専用のクラス」やその中でのメソッドを大量に書く必要があります。単純なクラスに対して使うと、かえってコードが複雑になり、管理が面倒になります。

### 2. オブジェクト生成のオーバーヘッド
直接 `new` する場合に比べて、Builderオブジェクトを経由するため、ごくわずかですがメモリや速度の面でコストがかかります（※現代のコンピュータではほとんど無視できるレベルですが、極限のパフォーマンスを求める場合は注意が必要です）。

### 3. 設計変更の際、修正箇所が増える
元のクラスに新しい項目を追加すると、Builderクラスの方にも同じ項目を追加し、セットするメソッドを書かなければなりません。二度手間になりやすいのが難点です。

# 適用パターン

| 状況 | Builderを使うべき？ |
| :--- | :--- |
| **コンストラクタの引数が4〜5個以上ある** | **強く推奨！** |
| **オプション（あってもなくても良い項目）が多い** | **最適！** |
| **一度作ったら中身を変えたくない（不変にしたい）** | **最適！** |
| **項目が2〜3個で、今後も増える予定がない** | **不要（普通のコンストラクタでOK）** |

> **例え話で理解：カスタムバーガーの注文**
> 
> マクドナルドのような「決まったセット」ではなく、サブウェイのように **「パンを選び、野菜を選び、ドレッシングを選び、最後に完成」** させるのがBuilderパターンです。
> 「野菜抜き」「ソース多め」といった細かいカスタマイズを、間違えることなく確実に形にすることができます。


## サンプルソース
```Java:Director.java
public class Director {
    private Builder builder;
    
    public Director(Builder builder) {   
    	this.builder = builder;
    }
    
    public Object constract(){
        builder.buiderPart1("X");
        builder.buiderPart2("Y");
        builder.buiderPart3("Z");
        return builder.getResult();
    }
}
```
```Java:Builder.java
public interface Builder {
    public void buiderPart1(String str);
    public void buiderPart2(String str);
    public void buiderPart3(String str);
    public Object getResult();
}
```
```Java:ConcreteBuilder01.java
public class ConcreteBuilder01 implements Builder{
    String str;
    @Override
    public void buiderPart1(String str) {
    	setString(str);
    }

    @Override
    public void buiderPart2(String str) {
    	setString(str);
    }

    @Override
    public void buiderPart3(String str) {
    	setString(str);
    }

    @Override
    public Object getResult() {
        return this.str;
    }
	
    private void setString(String str) {
        if (this.str == null) {
            this.str = str + ":";
        } else {   
    	    this.str += str + ":";
        }
    }
}
```
```Java:ConcreteBuilder02.java
import java.util.ArrayList;
import java.util.List;

public class ConcreteBuilder02 implements Builder{
    List<String> list = new ArrayList<>();
    
    @Override
    public void buiderPart1(String str) {
        list.add("01:" + str);
    }

    @Override
    public void buiderPart2(String str) {
    	list.add("02:" + str);
    }

    @Override
    public void buiderPart3(String str) {
    	list.add("03:" + str);
    }

    @Override
    public Object getResult() {
        return list;
    }
}
```
```Java:Client.java
public class Client {
    public static void main(String... args) throws Exception{
        Director director01 = new Director(new ConcreteBuilder01());
        System.out.println(director01.constract());
        
        Director director02 = new Director(new ConcreteBuilder02());
        System.out.println(director02.constract());
    }
}
```
