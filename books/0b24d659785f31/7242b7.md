### Oracle FreeSQL（Oracle Live SQL）のご紹介
SQLの学習を始めるにあたって、非常に便利なツールが「Oracle FreeSQL」です。その主なメリットと、本演習での活用方法について解説します。

#### **1. 環境構築の手間がいりません**
通常、SQLを学ぶには自分のPCにデータベースソフトをインストールするなどの複雑な設定が必要ですが、Oracle FreeSQLならその必要はありません。Webブラウザさえあれば、すぐに学習をスタートできます。

#### **2. サンプルデータが最初から用意されています**
本演習では、このサイトにあらかじめ用意されている「サンプルスキーマ（練習用のデータ）」を使用します。データ作成の手間を省き、純粋にSQLの記述練習に集中できる環境が整っています。

#### **3. Oracleアカウントについて**
Oracleアカウントを登録すると、自分専用のテーブルを作成したり、書いたスクリプトを保存したりといった、より自由度の高い環境を構築することも可能です。

※本演習を進める上では、独自の環境構築は行わず、標準のサンプルスキーマのみを使用します。

# 本演習で使用する4つのサンプルスキーマ

本プログラムでは、実務に近い多様なデータ操作を学ぶため、以下の4つのスキーマ（データの設計図）を使用します。

## 1. HRスキーマ：人事管理 (Human Resources)
主に **「人」と「組織」** に関するデータを扱います。
* 会社の従業員情報（氏名、職種、給与など）
* 部署や施設（オフィス所在地）などの管理

### HRスキーマのテーブル一覧

| テーブル名 | 役割・保持しているデータ |
| --- | --- |
| **`EMPLOYEES`** | 従業員の基本情報（氏名、給与、入社日、所属部署、上司など）。**HRスキーマの中心となるテーブル**です。 |
| **`DEPARTMENTS`** | 部署の情報（部署名、マネージャー、場所など）。 |
| **`JOBS`** | 職種の情報（職種名、その職種の最低・最高給与額など）。 |
| **`JOB_HISTORY`** | 従業員の異動履歴。過去にどの部署でどの職種についていたかを記録します。 |
| **`LOCATIONS`** | オフィスの詳細な住所情報。 |
| **`COUNTRIES`** | 国名と、所属するリージョンのID。 |
| **`REGIONS`** | 地域区分（Asia, Europe, Americas, Middle East and Africaなど）。 |


## 2. OEスキーマ：受注管理 (Order Entry)

**「在庫」と「注文」** の流れを管理するデータです。
* さまざまな販売チャネル（店舗やオンラインなど）を通じた製品の販売状況
* 製品の在庫管理
  
### OEスキーマのテーブル一覧
| テーブル名 | 役割・保持しているデータ |
| --- | --- |
| **`CUSTOMERS`** | 顧客の基本情報（名前、住所、電話番号、与信限度額など）。 |
| **`ORDERS`** | 注文の基本情報（注文日、注文ステータス、担当者、顧客IDなど）。 |
| **`ORDER_ITEMS`** | 注文の明細データ。どの注文で、どの商品を何個、いくらで販売したかを記録します。 |
| **`PRODUCT_INFORMATION`** | 商品カタログ。商品名、説明、カテゴリ、定価などの基本データ。 |
| **`INVENTORIES`** | 在庫情報。どの倉庫に、どの商品が何個あるかを管理します。 |


## 3. SHスキーマ：販売履歴 (Sales History)
**「分析」と「統計」** に特化したデータです。
* ビジネス上の意思決定（売上予測や傾向分析など）に役立つ事業統計データ
* 大規模な履歴データの集計

### SHスキーマのテーブル一覧
| テーブル名 | 役割・保持しているデータ |
| --- | --- |
| **`SALES`** | **中心となる売上データ**。いつ、どの顧客が、どの商品を、どのチャネルで購入したかという実績が記録されています。 |
| **`PRODUCTS`** | 商品の詳細データ（カテゴリ、サブカテゴリ、供給元など）。 |
| **`CUSTOMERS`** | 顧客の基本データ（氏名、住所、居住国など）。 |
| **`TIMES`** | 時間軸のデータ。日付、月、四半期、年、会計年度など、分析に必要な時間単位が細かく定義されています。 |
| **`CHANNELS`** | 販売経路（インターネット、電話、カタログ、店舗など）の区分。 |
| **`PROMOTIONS`** | プロモーション（広告キャンペーンなど）の情報。 |
| **`COUNTRIES`** | 国のデータ。顧客がどこの国の人か、どの地域に属するかを管理します。 |
| **`SUPPLEMENTARY_DEMOGRAPHICS`** | 標準的な販売データ（誰が何をいつ買ったか）だけでは見えてこない、**「顧客の背景（属性情報）」**　を補完するためのテーブルです。データ分析において、「なぜその顧客が離脱したのか？」を深掘りする際の説明変数として活用されます。|



## 4. COスキーマ：顧客注文 (Customer Orders)
**「小売アプリケーション」** を模した、シンプルで学習しやすいデータモデルです。
* 顧客情報、製品データ、店舗情報
* 注文履歴など、一般的なショッピングサイトの基本構成

これらのスキーマを使い分けることで、事務的なデータ更新から高度な集計分析まで、幅広いSQLスキルを習得することができます。

### COスキーマのテーブル一覧
| テーブル名 | 役割・保持しているデータ |
| --- | --- |
| **`CUSTOMERS`** | 顧客の連絡先やメールアドレスなどの基本情報。 |
| **`STORES`** | 商品を販売・発送する「店舗」の情報。 |
| **`PRODUCTS`** | 取り扱っている商品の名称や価格。 |
| **`ORDERS`** | 誰が、いつ、どの店舗で注文したかという履歴。 |
| **`ORDER_ITEMS`** | 各注文で「どの商品が何個売れたか」という明細。 |
| **`SHIPMENTS`** | 注文した商品がどこから発送され、現在どのような状態（配送中など）か。 |
| **`INVENTORY`** |  **「どの倉庫に、どの商品が、いくつあるか」** という、物流の「今」の状態を保持する。 |

# 演習問題のレベル設定について

当コンテンツでは、初心者から上級者まで幅広い層の方に学んでいただけるよう、多様な難易度の問題を用意しております。

特に初心者の方は、**「この問題は今の自分にとって、できて当然のレベルなのだろうか？」**と判断に迷うこともあるかと思います。そこで、私の主観ではありますが、各問題に以下の**4段階のレベル**を設定しました。学習を進める際の参考にしてください。

## 難易度レベルの一覧

| レベル | レベル感の目安 |
| --- | --- |
| **`LV1`** | **基礎中の基礎**です。SQLを扱うなら、必ず知っておくべき必須の内容です。 |
| **`LV2`** | **初級レベル**です。実務でも頻繁に使用するため、ぜひ習得しておきたい内容です。 |
| **`LV3`** | **中級〜上級レベル**です。初級者には難易度が高めですが、上級者ならスムーズに解ける内容です。 |
| **`LV4`** | **超上級レベル**です。SQLに精通した上級者でも、頭を悩ませるような難易度の高い問題です。 |

まずは **`LV1`** から着実にクリアしていき、自信がついたら上のレベルへ挑戦してみるのがおすすめです。たとえ **`LV3`** 以上でつまずいたとしても、それは高度な内容に挑戦している証拠ですので、安心してください！


# 演習の進め方
## 1. 各演習問題の構成

すべての問題は、以下の5つの要素で構成されています。学習のヒントが詰まっていますので、隅々までチェックしてみてくださいね。

* **問題**：解くべき課題の内容です。
* **期待する結果**：正解した場合に出力されるデータのイメージです。
* **解答例**：正解となるSQLスクリプトの例です。
* **解説**：なぜその書き方になるのか、ポイントを詳しく説明します。
* **参考リンク**（※ある場合のみ）：より深く知りたい方向けの補足情報です。

## 2. おすすめの学習ステップ

最短でスキルを身につけるために、以下のサイクルで進めることをお勧めします。

1. **まずは自力で挑戦**：問題と期待する結果だけを見て、自分の手でSQLを書いてみましょう。
2. **答え合わせと理解**：解答例を確認し、解説や参考リンクを読んで「なぜそうなるのか」を深く理解します。
3. **反復練習**：解答を見ずに自力で書けるようになるまで、繰り返し解きます。

## 3. 学習を継続するためのアドバイス

* **「考え抜く」と「あきらめる」のバランス**
最初は極力、解答を見ずに自分の頭で考えてみてください。試行錯誤こそが上達の近道です。ただし、**「これ以上考えても進めない」と感じた時は、迷わず解答を見てOKです！** 大切なのは、立ち止まって挫折しないことです。
* **「明日」の再チャレンジが効果的**
一度答えを見て理解した問題は、翌日に改めて「解答を見ずに」解いてみてください。記憶が定着し、自分の技術として定着しやすくなります。

すべての問題を「解答を見ずにスラスラ書ける」状態まで繰り返せば、あなたのSQLスキルは間違いなく上級レベルへと飛躍します。一緒に楽しみながら頑張りましょう！
