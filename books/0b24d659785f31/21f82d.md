## 解答例
```sql
SELECT 
    d.education,
    d.household_size,
    COUNT(DISTINCT churned_users.cust_id) AS churned_count,
    ROUND(AVG(churned_users.total_amount), 2) AS avg_last_year_sales
FROM sh.supplementary_demographics d
JOIN (
    -- 【副問合せ】離脱パターンの顧客を特定
    SELECT 
        cust_id, 
        SUM(amount_sold) as total_amount
    FROM sh.sales
    WHERE time_id BETWEEN DATE'2021-01-01'
                      AND DATE'2021-12-31'
    GROUP BY cust_id
    HAVING 
        -- 条件1：前半(H1)の売上が後半(H2)より多い（減少傾向）
        SUM(CASE WHEN time_id < DATE'2021-07-01' THEN amount_sold ELSE 0 END) >
        SUM(CASE WHEN time_id >= DATE'2021-07-01' THEN amount_sold ELSE 0 END)
        -- 条件2：2022年以降の購入が一度もない（完全離脱）
        AND cust_id NOT IN (
            SELECT DISTINCT cust_id 
            FROM sh.sales 
            WHERE time_id >= DATE'2022-01-01'
        )
) churned_users 
  ON d.cust_id = churned_users.cust_id
GROUP BY d.education, d.household_size
HAVING COUNT(DISTINCT churned_users.cust_id) > 15
ORDER BY churned_count DESC;
```

        -- パターン：2021年前半または後半のみが存在し、2022年以降(FUTURE)が存在しない

### 解答例
```sql
SELECT 
    OCCUPATION,
    COUNT(CUST_ID) AS CUST_COUNT,
    ROUND(AVG(YRS_RESIDENCE), 2) AS AVG_RESIDENCE,
    -- 全体の平均居住年数と比較
    ROUND(AVG(AVG(YRS_RESIDENCE)) OVER(), 2) AS TOTAL_AVG_RESIDENCE,
    -- 平均居住年数が多い順にランク付け
    RANK() OVER (ORDER BY AVG(YRS_RESIDENCE) DESC) AS RESIDENCE_RANK
FROM SH.SUPPLEMENTARY_DEMOGRAPHICS
GROUP BY OCCUPATION
ORDER BY RESIDENCE_RANK;
```

### 4. 応用編：INSTRとの組み合わせ　→　他へ
「カンマの前の文字だけを抜く」といった、場所が動的に変わる場合は、以前お話しした `INSTR` 関数（文字の場所を探す関数）と組み合わせて使います。

```sql
-- '@' より前のユーザー名部分だけを抜き出す
SELECT 
    SUBSTR(email, 1, INSTR(email, '@') - 1) 
FROM 
    users;
```
